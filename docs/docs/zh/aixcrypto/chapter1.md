# AI x Crypto 的核心框架

本节讨论人指导AI x Crypto领域发展方向的框架。我们将首先分析两个案例，然后讨论一般原则。

## 2.1 介绍性案例、研究

### 案例研究：Privasea

> Privasea 是一家致力于在训练AI模型的过程中保持所有人数据隐私的公司。他们采用一种被称为完全同态加密（Fully Homomorphic Encryption，FHE）的方法，使得能够在加密的数据上使用AI模型。例如，当用户向ChatGPT发出指令时，该指令被发送到OpenAI服务器。OpenAI能够查看该指令，将其提供给模型，然后返回输出结果。通过 FHE，你可以创建一个接受 "加密指令"的 ChatGPT 版本。因此，当我给 ChatGPT发送指令时，OpenAI永远不会看到我实际的指令，只会看到其加密版本。这在个人隐私方面有用武之地，并且也对大型企业也至关重要。对冲基金不允许员工使用 ChatGPT 等工具，因为这会泄露商业机密（只允许使用内部模型）。而针对AI模型的FHE等技术可以通过加密指令解决这个问题。

在这个例子中，Crypto正在帮助改进AI产品。更确切的来说，使用了一种加密方法来改变AI模型的训练方式。这就产生了新的功能：一个可以接受加密指令的AI模型。因此，由于采用了加密工具，AI的可能性空间现在变得更大了。

现在，让我们来看第二个案例。

### 案例研究：Dorsa

> Dorsa是一家区块链安全领域的初创公司。他们利用AI模型对智能合约进行审计和监控。他们开发了多种工具，包括强化学习智能体，以全面测试智能合约的安全性。Dorsa开发的工具可以为开发和审计人员提供帮助。因此，Dorsa利用AI模型来提高智能合约的安全性和可靠性。

在上面的例子中，AI正在协助改进一款Crypto产品。具体而言，Dorsa采用了一个链下的AI模型，以使得构建安全的智能合约的过程更加迅速和高效。这个AI还可能发现传统方法无发现的漏洞，从而预防之后的黑客攻击。需要注意的是，AI并不会带来新的功能，编写安全的智能合约一直都是可能的。AI的作用是使这一过程更快、更便宜。

现在，在讨论一般原则之前，让我们先看最后一个案例。

### 案例研究 LoverGPT

> LoverGPT 是一款 Web2 AI 女朋友应用。他们并不会审查裸露的聊天内容或图像，因此可能会面临无法接入Visa和Mastercard支付方式的问题。为了解决这个问题，他们开始接受加密货币的支付方式。然而产品的其他部分并没有使用任何区块链技术。
>
> 这属于AI x Crypto吗? 不属于。从技术上来讲，这个产品同时有一个AI组件（聊天机器人）和一个Crypto组件（加密货币支付）。但是，两者之间并没有实质性的交互。如果这家公司销售大麻并接受加密货币支付，也并不会有太大区别。加密货币为那些可能受到 Visa 审查的公司提供了一种解决新的方案，但这家公司并不属于AI x Crypto 的范畴。

上面的例子展示了一个包含Crypto和AI组件的产品。虽然这是一个Crypto技术用例，但它并不属于一个AI x Crypto的项目。AI和Crypto可以是相辅相成的技术。AI有助于改进crypto栈，而crypto也可以有助于改进改进AI栈。这可以通过Privasea（FHE-ML）和Dorsa（智能合约安全的AI模型）的例子来说明。然而，在LoverGPT的例子中，Crypto和AI只是简单地共存。

**Crypto和AI是相辅相成的技术，他们之间可以相互增强。实现这一潜力的项目被归类为 AI x Crypto 项目。**

## 2.2 AI 与 Crypto 相互促进

回顾上面的两个例子。

在 Privasea 中，利用FHE（一种加密方法）来创建能够保持用户数据私密的AI模型。使用加密的原语来改进AI栈（模型的训练方式），因此Crypto正在为AI提供帮助。

与此同时，在Dorsa中，AI模型用来确保智能合约的安全性。AI工具被用来改进Crypto栈的一个方面（开发安全的智能合约）。因此，AI正在帮助Crypto。

在评估AI x Crypto的项目时，我们需要对两者进行区分：是Crypto用来帮助AI？还是AI被用来帮助Crypto？

这个简单的问题可以帮助我们发现当前用例的一个重要方面。这里要解决的关键问题是什么？在 Dorsa 的案例中，我们期望的结果是一个安全的智能合约。这可以由熟练的开发者来完成，但 Dorsa 恰好利用AI提高了这一过程的效率。不过，从根本上说，我们只关心智能合约的安全性。一旦明确了关键问题，我们就能确定是AI在帮助Crypto，还是Crypto在帮助AI。在某些情况下，两者之间并不存在有意义的交互（例如 LoverGPT）。

## 2.3 产品定位

Crypto帮助AI，这可以通过多种方式实现，具体取决于Crypto部分与AI栈的交互方式。AI栈包括用于创建AI模型的步骤：数据处理、模型训练和推理。Crypto可以通过以下两种方式对AI提供帮助：

- 内部：在这种情况下，Crypto用于对AI栈进行内部更改（例如更改模型的训练方式）。例如上文提到的完全同态加密（FHE），这可以让我们在训练AI模型的同时保持数据的私密性（加密）。我们从根本上改变了AI模型的训练方式，AI栈中直接内置了Crypto部分。这就是经过改进的AI栈。

- 外部：这里使用Crypto来帮助AI栈，而不对其进行内部更改。例如Bittensor 鼓励用户贡献数据。这些数据可用于训练AI模型。在这种情况下，我们并没有改变模型训练或推理的任何方面。AI栈没有发生任何内部变化,代币奖励机制只是锦上添花。

![图四](/assets/20240219132408.png)

<center>图 4：Crypto帮助AI的两种方式示意图。AI栈包含创建人工智能模型所需的步骤，例如训练模型。当Crypto被用于改变训练过程时，这属于内部的。如果在不改变AI栈本身的情况下使用Crypto来帮助改进产品，这就是外部的。</center>

同样，AI也可以通过这两种方式帮助Crypto：

- 内部：AI是Crypto栈的内部组件。将AI部署在链上，可以直接与Crypto栈内部相连接。例如：管理 DAO 的链上AI agent。这种AI不仅仅是协助Crypto，而是直接构建在技术栈内，使 DAO 能够正常运行。

- 外部：AI存在于Crypto栈的外部。AI用于支持Crypto栈，而不对其进行内部更改。例如Dorsa 使用AI模型来确保智能合约的安全。该AI存在于链下，并且是一个外部工具，可以加速编写安全智能合约的过程以及降低成本。

基于上述区别，我们现在可以将原始框架扩展到 2x2 矩阵。AI可以通过两种方式帮助Crypto：栈的内部和外部。这同样适用于Crypto帮助AI。这就为我们提供了一个框架的 4 个支柱，如下图所示。

## 2.4 确定瓶颈

在这四个领域中，我们都看到了不同的挑战。一般来说，内部变更带来的技术挑战通常比外部变更更大。如果我们采用全同态加密（FHE）或者零知识证明（ZK）修改AI栈，我们需要深厚的密码学专业知识以及AI领域的经验。因此，Modulus、EZKL、Zama 和 Privasea 等公司需要解决深层次的技术问题，以改进和扩展其解决方案。在Crypto栈中集成AI的公司面也同样临着极具挑战性的工程问题。允许用户在智能合约中集成AI,必须要对Crypto栈和AI模型的工作原理有深入的了解。因此，像Ritual 和 Ora这样的公司需要解决这些复杂的工程问题。

为栈提供外部支持也会遇到不同的瓶颈，但这些瓶颈通常在技术上并不那么复杂。例如，为AI agent提供支付渠道并不需要面对如何训练AI模型的艰难挑战。在AI在外部帮助Crypto的情况下，一个限制因素是在Crypto领域中AI人才的稀缺。

为 DeFi LLama 构建一个 ChatGPT 插件，对AI工程师来说技术上并不复杂。但是，在Crypto领域，这样的工程师并不多。

在下表中，我们总结了在这些领域中推进的主要瓶颈：

| 说明                 | 创新的主要瓶颈                                                     |
| ------------------ | ----------------------------------------------------------- |
| Crypto 在内部帮助 AI    | 需要解决基本的深层技术难题（例如，Scaling ZKML、Scale homomorphic encryption） |
| Crypto 从外部帮助 AI    | 良好的经济模型（例如，如何激励数据收集，或奖励为 AI 网络工作贡献 GPU 的用户）                 |
| AI 在内部帮助 Crypto    | 工程挑战。相对于 Crypto 栈而言，AI 工具已经成熟。添加 AI 需要解决复杂的工程问题。            |
| AI 为 Crypto 提供外部帮助 | Crypto 领域 AI 人才稀缺                                           |

## 2.5 衡量效用

矩阵上四个区域中每个区域的产品都有不同的价值。对堆栈进行内部修改会产生修改后的堆栈。这通常会带来新的功能。如果在加密货币堆栈内部集成了人工智能，这就意味着智能合约开发者可以使用人工智能模型。这就增加了可能的空间，并有可能带来创新。这同样适用于加密货币在内部帮助人工智能。

在矩阵上的这四个领域中，每个领域的产品都有不同的价值。如果AI对Crypto有内部帮助，那就意味着Crypto的开发人员有将更强大的工具。这增加了可能的空间，并可能带来创新。这同样适用于Crypto在内部帮助AI。简而言之，如果是内部帮助，那么栈本身就会发生变化。这就打开了全新的可能性。

![图六](/assets/20240219135304.png)

<center>图 6：在Crypto栈中加入AI为开发人员提供了新的超能力。这增拓展了技术可能性，有望带来重大创新。</center>

就AI对Crypto的外部帮助而言，AI更有可能只是改进现有产品。例如，使用AI模型编写智能合约可能比以前更快、更便宜, 以及提高其安其全性。但它不太可能激发任何创新。这同样适用于Crypto在外部帮助AI。代币激励可以用于AI栈，但这本身不太可能改变我们训练AI模型的方式。

简而言之，一方面，集成在栈内部的技术更有可能会带来新的功能。另一方面，在栈外部使用的技术可能会提高现有产品的效率。

## 2.6 评估上升空间

我们可以根据上述信息评估项目的上升空间。不过，这也有主观因素。估算技术的潜在上升空间需要考虑诸多因素。

需要考虑的一个因素是这是与 Web2、Web3 还是两者都相关？完全同态加密（FHE）可用于存在于AI模型的任何环境中。这对用于所有领域的AI模型都很有用。另一个因素是：这是否能提高效率或有能力实现全新功能（见第 2.5 节）？

一旦确定了某项潜在技术的主观上升空间，估算该技术成熟的时间跨度就变得很有帮助。这个估算可以基于对当前的进展和该领域的瓶颈（见第 2.4 节）来完成。将这两个因素结合起来，就可以在图表中对不同项目进行评估。

## 2.7 了解复杂产品

有些产品可以同时涵盖四个支柱，而不仅仅是其中之一。下面是几个例子。

通过 Flock.io，您可以进行联邦学习，并通过代币经济模型奖励用户。通过联邦学习，我们可以建立一个AI模型，其中每个人的数据都是私有的。在这里，Crypto 在内部帮助 AI，因为 AI 栈（模型训练）发生了变化。

上面的 tokenomics 模型不会改变AI模型的训练方式。它被用作一种激励机制，以奖励用户的贡献。这是Crypto对AI的外部帮助。

Crypto如何在内部帮助AI引入了一种新能力：AI模型可以在去中心化网络中进行训练，同时保持数据的私密性。Crypto 在外部对AI的帮助并没有引入新的功能。代币被用于激励用户为该网络做出贡献，从而提高系统的效率。
