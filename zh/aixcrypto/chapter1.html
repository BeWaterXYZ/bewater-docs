# 人工智能 x 密码的核心框架

在本节中，我们将讨论人工智能 x Crypto 领域的导航框架。我们将从两个人工智能 x Crypto 项目的案例研究开始。在分析这些项目之后，我们将讨论一般原则。

# 2-1 介绍性案例、研究

# 案例研究：Privasea

Privasea 是一家公司，其使命是让人工智能模型的训练方式能够保护每个人的数据隐私。他们使用一种名为全同态加密（FHE）的方法。这样就能在加密数据上使用人工智能模型。例如，当用户当前提示 ChatGPT 时，该提示会被发送到 OpenAI 服务器。OpenAI 可以看到该提示，并将其反馈给模型和返回输出。通过 FHE，您可以创建一个接受 "加密提示 "的 ChatGPT 版本。因此，当我提示 ChatGPT 时，OpenAI 永远不会看到我的实际提示，只会看到其加密版本。这在个人隐私方面也有用武之地。此外，这对大型企业也至关重要。对冲基金不允许员工使用 ChatGPT。

在这个例子中，加密技术正在帮助改进人工智能产品。具体来说，一种加密方法被用来改变人工智能堆栈（人工智能模型的训练方式）。这就产生了新的功能：一个可以接受加密提示的模型。因此，由于采用了加密工具，人工智能的可能性空间现在更大了。如下图所示。

如上图所示，使用加密技术对人工智能堆栈进行内部更改，可产生新的功能。例如，FHE 允许我们使用加密提示。

# 案例研究：多尔萨

Dorsa 是一家区块链安全初创公司。他们使用人工智能模型来审核和监控智能合约。他们开发了各种工具，包括强化学习代理，以广泛测试智能合约的安全性。Dorsa 开发的工具可以帮助开发人员和审计人员。因此，Dorsa 正在利用人工智能模型来提高智能合约的安全性和可靠性。

在上面的例子中，人工智能正在帮助改进一种加密产品。具体来说，链外人工智能模型被用来使创建安全智能合约的过程更快、更便宜。这种人工智能还可能发现传统方法无法发现的漏洞，从而防止未来的黑客攻击。请注意，人工智能并不会带来新的能力。编写安全的智能合约总是可能的。人工智能的作用是使这一过程更快、更便宜。

现在，让我们先看最后一个案例，然后再讨论一般原则。

# 案例研究 LoverGPT

LoverGPT 是一款 Web2 AI 女朋友应用程序。他们不审查露骨的聊天内容或图片，因此可能会面临维萨卡和万事达卡的问题（类似于这家公司被审查）。为了解决这个问题，他们接受加密货币支付。产品的其他部分没有使用区块链。

这是人工智能 x 加密货币吗？从技术上讲，有一个人工智能组件（聊天机器人）和一个加密货币组件（加密货币支付）。但是，这两者之间并不存在有意义的互动。如果这家公司销售大麻并接受加密货币支付，那也不会有什么不同。加密货币为那些可能受到人工智能和加密货币的结合往往流于表面，没有实际价值。

上面的例子展示了一个包含加密和人工智能组件的产品。虽然这是一个加密货币用例，但它并不是一个人工智能 x 加密货币项目。人工智能和加密技术可以是相互支持的技术。人工智能可以帮助改进加密堆栈，而加密也可以帮助改进人工智能堆栈。Privasea（FHE-ML）和 Dorsa（智能合约安全的人工智能模型）的例子就说明了这一点。不过，在 LoverGPT 的例子中，加密技术和人工智能只是共存而已。

加密技术和人工智能是相辅相成的技术。每种技术都能相互促进。实现这一潜力的项目被归类为 AI x Crypto 项目。可以通过本节提供的框架来了解和评估这些项目。

# 2.2 人工智能与加密货币互相帮助

回顾上面的两个例子。在 Privasea 中，FHE（一种加密方法）被用来创建能够保护用户数据隐私的人工智能模型。密码学原理被用来改进人工智能堆栈（模型的训练方法），因此密码学正在帮助人工智能。

与此同时，在 Dorsa，一个人工智能模型正被用于确保智能合约的安全。在这里，人工智能工具被用来改进加密堆栈的一个方面（开发安全的智能合约）。因此，人工智能正在帮助加密货币。

在评估人工智能与加密货币交叉项目时，这就给我们带来了一个重要的区别：加密货币是用来帮助人工智能的？还是人工智能被用来帮助加密货币？

这个简单的问题可以帮助我们发现当前用例的一个重要方面。这里要解决的关键问题是什么？在 Dorsa 的案例中，我们希望得到的结果是一份安全的智能合约。这可以由熟练的人工来开发，但 Dorsa 借助人工智能使这一过程更加高效。不过，从根本上说，我们只关心安全的智能合约。一旦明确了关键问题，我们就能确定是人工智能在帮助加密货币，还是加密货币在帮助人工智能。在某些情况下，两者之间并不存在有意义的互动（例如 LoverGPT）。

下表提供了每个类别中的几个例子。

加密货币助力人工智能 人工智能助力加密货币
1. 使用 FHE 训练人工智能模型，同时保持用户数据的私密性 1. 收集和标注数据集的代币激励机制
2. 使用人工智能模型来保护、审计和监控智能合约 2. 高效搜索链上数据的人工智能模型
3. 用于人工智能模型训练的联合学习 3. 人工智能代理的支付轨道
4. 用于创建数据分析仪表板的 LLM 4. 基于意图的交易平台
5. 可验证推理的零知识证明 5. 分散式 GPU 网络
6. 管理 DAO 的人工智能代理 6. 为连锁游戏定制机器人

总结一下：人工智能和加密货币是相辅相成的技术。要评估任何产品，关键是要了解是人工智能在帮助加密货币，还是加密货币在帮助人工智能。

# 2.3 产品 定位

加密技术可以帮助人工智能。不过，这可以通过多种方式实现，具体取决于加密组件与人工智能堆栈的交互方式。人工智能堆栈包括用于创建人工智能模型的步骤：数据处理、模型训练和推理。加密技术可以通过两种方式帮助人工智能：

  • 内部：在这里，加密技术用于对人工智能堆栈进行内部更改（例如，更改模型的训练方式）。例如完全同态加密（FHE）可以让我们在训练人工智能模型的同时保持数据的私密性（加密）。在这里，我们从根本上改变了人工智能模型的训练方式。一个加密图形组件被直接内置到人工智能堆栈中。这就是经过改进的人工智能堆栈。

  • 外部：这里使用加密来支持人工智能堆栈，而不对其进行内部更改。例如Bittensor 鼓励用户贡献数据。这些数据可用于训练人工智能模型。在这里，我们没有改变模型训练或推理的任何方面。人工智能堆栈没有发生任何内部变化。代币奖励只是锦上添花。

人工智能堆栈包含创建人工智能模型所需的步骤，例如训练模型。当加密技术被用于改变训练过程时，这是内部过程。如果加密技术用于帮助改进产品，而不改变人工智能堆栈本身，这就是外部加密技术。

同样，人工智能可以从两个方面帮助加密货币。我们可以将加密货币堆栈视为一个技术蛋糕。人工智能可以通过同样的两种方式帮助加密货币：

  • 内部：人工智能是加密堆栈的内部组件。人工智能生活在链上，直接与加密堆栈内部的组件对接。例如：管理 DAO 的链上人工智能代理。这种人工智能不只是协助加密堆栈。它直接构建在技术堆栈内部，使 DAO 能够运行。

  • 外部：人工智能是加密堆栈的外部设备。人工智能用于支持加密堆栈，而不对其进行内部更改。例如Dorsa 使用人工智能模型来确保智能合约的安全。人工智能生活在链外，是一种外部工具，用于使编写安全智能合约的过程更快、更便宜。

基于上述区别，我们现在可以将原始框架扩展到 2x2 矩阵。人工智能可以通过两种方式帮助加密算法：堆栈内部和堆栈外部。这同样适用于密码帮助人工智能。这就为我们带来了框架的 4 个支柱，如下图所示。

# 2.4 瓶颈

在这四个领域中，我们都看到了不同的挑战，因此你必须谨慎地选择你的战斗。一般来说，内部更改比外部更改带来更多的技术挑战。如果我们用全同态加密（FHE）或零知识证明（ZK）来修改人工智能堆栈，我们需要深厚的密码学专业知识以及人工智能经验。因此，Modulus、EZKL、Zama 和 Privasea 等公司需要解决深层次的技术问题，以改进和扩展其解决方案。在加密堆栈中集成人工智能的公司面临着非常具有挑战性的工程问题。允许用户在智能合约中集成人工智能需要对加密货币堆栈和人工智能模型的工作原理有深入的了解。因此，像 Ritual 和 Ora 这样的公司可以解决这些复杂的工程问题。

外部支持堆栈有不同的瓶颈，而这些瓶颈通常在技术上并不那么复杂。例如，为人工智能代理提供支付轨道并不需要解决如何训练人工智能模型的难题。人工智能对外帮助加密货币的一个限制因素是加密货币领域人工智能人才的稀缺。为 DeFi LLama 构建一个 ChatGPT 插件，对于 AI 工程师来说技术上并不复杂。但是，在加密货币领域，这样的工程师并不多。

在下表中，我们总结了在这些领域推进产品的主要瓶颈。

说明 创新的主要瓶颈
加密货币在内部帮助人工智能 需要解决基本的深层技术难题（如扩展 ZKML、扩展同态加密等）
加密货币从外部帮助人工智能 良好的经济模型（例如，如何激励数据收集，或奖励为网络贡献 GPU 的用户）
人工智能在内部帮助加密货币 工程挑战。相对于加密堆栈而言，人工智能工具已经成熟。添加人工智能需要解决复杂的工程问题
人工智能为加密货币提供外部帮助 加密领域人工智能人才稀缺

# 2.5 评估实用性

矩阵上这四个领域中的每一个产品都能以不同的方式发挥价值。如果人工智能对加密货币有内部帮助，那就意味着开发人员可以使用新的强大工具。这增加了可能的空间，并有可能带来创新。这同样适用于加密货币在内部帮助人工智能。简而言之，如果帮助是内部的，那么堆栈本身就会发生变化。这就打开了可能的领域。区块链开发者可以从智能合约内部访问人工智能，这很可能会带来新的能力和创新。同样，在人工智能堆栈内部使用加密方法也会带来新的功能（如 Privasea 中的加密提示）。

在加密堆栈中加入人工智能为开发人员提供了新的能力。这增加了可能的表面积，并可能带来重大创新。就人工智能对加密货币的外部帮助而言，人工智能更有可能只是改进现有产品。例如，使用人工智能模型编写智能合约可能会比以前更快、更便宜。它还可能提高安全性。不过，它不太可能引发全新的创新。这同样适用于加密货币在外部帮助人工智能。代币激励可以用于人工智能堆栈，但这本身不太可能改变我们训练人工智能模型的方式。

简而言之，集成在堆栈内部的技术可能会带来新的能力。另一方面，在堆栈外部使用的技术更有可能提高现有产品的效率。

# 2.6 评估 向上

可以根据上述信息评估项目的上升空间。不过，这也有主观因素。估算技术的潜在上升空间需要考虑许多因素。

需要考虑的一个因素是这与 Web2、Web3 还是两者都相关？完全同态加密（FHE）适用于任何存在人工智能模型的环境。这对用于所有领域的人工智能模型都很有用。另一个因素是：这是否能提高效率或实现新功能。

一旦确定了某项潜在技术的主观上升空间，估算出该技术成熟的时间跨度就很有帮助。这可以根据当前的进展情况和该领域的瓶颈来进行（见第 2.4 节）。这两个因素的组合可以显示在图表上，以评估不同的项目。

# 2.7 了解复杂的产品

有些产品可以同时具备四大支柱，而不仅仅是其中之一。下面是几个例子。

通过 Flock.io，您可以进行联合学习，并以代币经济模型奖励用户。通过联合学习，我们可以建立一个人工智能模型，其中每个人的数据都是私有的。在这里，加密货币在内部帮助人工智能，因为人工智能堆栈（模型训练）发生了变化。

上面的 tokenomics 模型不会改变模型的训练方式。它是一种激励机制，用于奖励用户的贡献。这是加密货币对人工智能的外部帮助。

请注意加密货币如何在内部帮助人工智能引入了一种新的能力：模型可以通过去中心化网络进行训练，同时保持数据的私密性。然而，加密货币在外部帮助人工智能并没有引入新的能力。代币用于激励用户为该网络做出贡献。用户已经可以获得法定货币补偿。用加密货币激励用户更有优势，从而提高了系统的效率。但是，它并没有引入新的功能。

在这里，ZK 被用来改变人工智能堆栈。由此产生的人工智能模型现在具有可验证的推理能力，而通常情况下是不具备这种能力的。在这里，人工智能堆栈内部使用了加密技术，使模型变得更好。
